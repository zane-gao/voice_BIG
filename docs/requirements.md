# 需求整理

## 目标

实现一个“语音 + 草图”多模态科研方法图生成系统。用户在浏览器中画草图、录音或上传音频，系统自动生成结构化方法图，输出 Mermaid、SVG 预览和 JSON。

## 输入输出

输入：

- 手绘草图：浏览器 canvas 导出的 PNG。
- 语音：浏览器录音 WebM 或上传音频文件。
- 可选人工文本：用户可直接填写或修改转写文本。
- 图方向：`LR` 或 `TD`。

输出：

- `transcript`：最终使用的语音转写文本。
- `graph`：包含标题、摘要、节点、边、告警和置信度的结构化 JSON。
- `mermaid`：可编辑的 Mermaid flowchart。
- `image_b64`：豆包草稿图或 OpenAI 终稿图的 base64 图像。
- `timings_ms`：ASR、图结构化、Mermaid 转换和总耗时。

## 验收标准

- 浏览器可完成“画草图 + 录音/上传音频 + 生成方法图”的闭环。
- 后端在无 API key 时仍能使用 mock 模式演示。
- 输出图中没有悬空边、重复边和非法节点 id。
- 停止画图或编辑文本 10 秒后可以自动生成低清草稿图。
- 点击终稿按钮可以生成或 mock 回退一张高清终稿图。
- 文档和报告可说明系统架构、实验设计、指标和局限。

## 风险

- OpenAI 账号可能不支持默认模型，可通过 `.env` 切换模型。
- 浏览器录音依赖 HTTPS 或 localhost 权限。
- Mermaid CDN 离线时图形预览不可用，但源码仍可下载。
