# 一、科研侧的总定位要改成这样

不要说：

> 我们研究了语音生成知识图。

这个说法太浅，也容易被理解成“语音转文字 + 大模型画图”。

应该说：

> **我们研究的是“语音引导的知识草图结构化”问题：给定一个手绘知识草图和同步语音描述，系统需要恢复一个可编辑、可评测、可渲染的结构化知识图。**

这个定义更深，因为它强调了三件事：

第一，输入不是单一语音，而是 **草图 + 语音** 。

第二，输出不是图片，而是 **结构化知识图** 。

第三，目标不是好看，而是 **节点、边、方向、语义标签、布局关系都尽可能正确** 。

你们报告里其实已经有非常明确的问题定义：给定手绘知识草图、语音转写和目标知识图，系统需要输出可编辑知识图及其矢量化渲染；节点包含标签、语义类型、解释文本和置信度，边包含起点、终点、关系类型、关系标签和置信度。

所以科研侧第一句话建议是：

> **我们没有把这个任务简单看作“语音转文本”或“图像生成”，而是把它定义为一个多模态知识结构恢复问题。**

# 二、研究动机要讲成“模态互补”

这一部分要讲深刻，必须讲清楚：**为什么一定要语音 + 草图？**

你可以这样讲：

在科研讨论、课堂板书、组会推导和论文方法设计中，人类通常不是用完整文本表达想法，而是采用一种非常自然的混合表达方式：一边画框图和箭头，一边口头解释模块名称、关系方向、设计动机和约束条件。

这就导致两个单模态都有缺陷：

**只看草图的问题：**

草图能表达空间布局，比如上下游模块、并行分支、反馈闭环，但草图中的文字可能潦草、缩写不规范，箭头可能断裂，模块含义可能模糊。

**只听语音的问题：**

语音能补充概念名称、先后顺序、因果关系和解释语义，但语音没有二维布局信息，很难知道哪些模块并列、哪些模块属于同一层级、哪些是主干、哪些是辅助说明。

所以这件事的科学本质是：

> **草图提供空间先验，语音提供语义锚点；二者共同约束最终的知识图结构。**

你们报告中的假设也正好对应这个逻辑：H1 语音反馈可以补全草图中未写全或难以识别的节点标签；H2 口头叙述顺序可以纠正草图中模糊或断裂的边方向；H3 显式约束图编辑路径可以降低用户修改成本，并提供路径能量诊断。

这一段是科研侧的核心深度。因为它说明你们不是为了“功能丰富”而加入语音，而是因为语音在理论上提供了草图缺失的约束。

# 三、相关工作不要泛泛讲，要讲出“空白”

你可以把相关工作分成三条技术路线，然后指出你们的位置。

## 1. 草图到结构图

这一类研究关注如何把手绘草图变成结构化图。比如 SketchAgent 认为手绘草图是自然高效的思想表达媒介，但自由手绘草图缺乏结构约束和语义精度，因此从草图到机器可读图仍然困难；该工作还提出了 Sketch2Diagram Benchmark，覆盖流程图、有向图、模型架构等八类图示，并包含超过 6000 个高质量样例。

这类工作的不足是：

> 它们主要关注视觉草图本身，较少利用同步语音作为额外语义反馈。

也就是说，它解决的是  **Sketch → Diagram** ，而你们关注的是  **Sketch + Speech → Structured Knowledge Graph** 。

## 2. 图像到可编辑图代码

Flowchart2Mermaid 这类工作关注把静态流程图图像转成 Mermaid 代码，因为静态图难以编辑和复用；它强调输出结构化、可版本控制的文本表示，并评测结构准确性、流程正确性、语法合法性和完整性。

这类工作的启发是：

> 输出不应该只是图片，而应该是可编辑的结构化表示。

这和你们系统输出 Mermaid、SVG、JSON 的设计高度一致。你们 README 也写明系统会输出 Mermaid、SVG 预览、JSON、AI 生成论文风格图像和终稿图语音讲解 WebM。

但它的不足是：

> 它更多处理已有流程图图片，而不是人类实时草图与口头讲述结合的开放式科研草图。

## 3. 薛定谔桥 / 图结构变换

薛定谔桥的价值在于，它不是简单地从噪声生成数据，而是研究两个分布之间的动态迁移。Diffusion Schrödinger Bridge Matching 将生成过程与动态传输问题联系起来，强调 SB 可以恢复熵正则化的最优传输形式。 DDSBM 进一步把 SB 扩展到高维离散状态空间，并用于图变换；其节点和边的独立修改可以解释为基于图编辑距离代价的熵正则最优传输。

这就给你们提供了理论支点：

> 我们的目标不是在像素空间直接生成一张图，而是在离散图结构空间中，把草图/语音得到的粗糙候选图逐步编辑成标准知识图。

这就是你们 MMSB-Graph 的合理性。

# 五、方法讲法：从“一步生成”改成“图编辑路径”

这是科研侧最重要的深度。

普通做法可能是：

> 草图 + 语音 → 大模型 → Mermaid / 图像。

这个是 one-shot generation，最大问题是不可解释：错了不知道为什么错，也不知道错在哪一步。

你们的方法应该讲成：

> 我们不直接让模型一步生成最终图，而是把过程拆成三个中间图，再通过图编辑桥逐步修复。

三个中间图是：

Gs:草图单模态图G_s: \text{草图单模态图}**G**s:**草图单模态图**
它主要来自草图，保留框、箭头、二维布局，但节点标签可能缺失，边方向可能错误。

Gu:语音单模态图G_u: \text{语音单模态图}**G**u:**语音单模态图**
它主要来自语音，节点标签和顺序关系较强，但缺少空间布局。

Gf:早期融合图G_f: \text{早期融合图}**G**f:**早期融合图**
它把草图和语音的信息合并，作为图编辑桥的初始候选图。

然后你们用图编辑桥从 GfG_f**G**f 迁移到目标知识图 G⋆G^\star**G**⋆。报告里明确说：草图单模态图强调框、箭头和二维布局，但允许节点标签缺失；语音单模态图强调口头提到的模块和先后关系，但没有可靠布局；早期融合图合并二者作为桥过程初始状态。

更具体一点，你们的图编辑桥允许四类操作：

1. 删除多余节点；
2. 插入缺失节点；
3. 删除错误边；
4. 插入目标边。

报告里也写明，节点编辑权重高于边编辑，因为错误节点通常需要更多人工修改。

这部分可以这样讲：

> **我们关心的不只是最后结果对不对，还关心系统是通过什么编辑路径到达最终结果的。因为在真实应用中，用户修改成本本质上就是图编辑成本。**

这句话很关键，能把“路径能量”讲得非常真实。

# 六、为什么叫 MMSB-Graph，要讲出“不是硬套名词”

你们要避免老师觉得“薛定谔桥”是强行包装。可以这样解释：

在经典生成模型里，常见做法是从噪声分布生成数据分布。但我们的源不是纯噪声，而是一个已经有结构但不完整的图：它来自草图、语音和早期融合结果。

所以我们的问题更像：

粗糙结构图分布→标准知识图分布\text{粗糙结构图分布} \rightarrow \text{标准知识图分布}**粗糙结构图分布**→**标准知识图分布**
而不是：

随机噪声→目标图像\text{随机噪声} \rightarrow \text{目标图像}**随机噪声**→**目标图像**
因此，薛定谔桥或者最优传输的思想更合适：我们希望在保留源图信息的同时，用最小必要编辑把它迁移到目标图。

你们报告中的优化目标写成了：在离散图空间 G\mathcal G**G** 上寻找从源图到目标知识图的随机编辑路径 G0:TG_{0:T}**G**0**:**T****，并同时考虑草图约束、语音约束和融合约束。

用答辩话术讲就是：

> **MMSB-Graph 的核心不是做像素级扩散，而是把知识图看成离散状态，把节点和边的修改看成图编辑动作，把语音和草图看成多边际约束，从而获得一个可解释的结构恢复路径。**

这句话很重要。

# 七、代价函数要讲具体

不要只放公式，要解释每一项在真实场景里对应什么。

你们报告里定义了单步编辑代价，包括图编辑距离、标签语义距离、边方向代价、草图保持项、语音覆盖奖励。

你可以这样讲：

c(Gt,Gt+1)c(G_t, G_{t+1})**c**(**G**t,**G**t**+**1****)
不是一个抽象数学符号，而是模拟用户真实修改成本：

| 代价项       | 含义                   | 真实解释                                 |
| ------------ | ---------------------- | ---------------------------------------- |
| 图编辑距离   | 节点/边增删改          | 用户需要手动改多少结构                   |
| 标签语义距离 | 节点名称是否对         | “卷积核”不能识别成“卷积层参数”或乱码 |

| 语音覆盖奖励 | 是否覆盖口头提到的信息 | 用户说了“评测指标”，图里不能漏掉       |

> 如果最终图看起来对，但路径能量很高，说明系统其实做了大量强行修正，初始解析质量不好；如果路径能量低但最终错误多，说明系统太保守，没有充分利用语音和目标约束。

这就是你们比普通“最终准确率评测”更深的一点：你们不仅评估结果，还评估恢复过程。

# 九、实验协议要讲真实，不要吹大

你们现在实验最大的风险是：结果表里 MMSB-Graph 在可控设定下达到 1.000，容易被老师质疑“是不是设计出来的”。

所以正确讲法是：

> 我们把实验分成两层：第一层是合成条件图可控验证，用来验证评测链路和图编辑桥是否可计算；第二层是真实模型实验，后续需要接入真实 ASR/VLM 输出继续测试。当前课程阶段，我们主要完成第一层。

报告也已经明确说明，合成条件图实验不调用外部视觉语言模型或语音识别模型，因此不能代表真实模型性能，只用于验证指标链路和图桥行为。

这句话必须主动说。因为主动承认边界，反而显得真实。

实验对照可以讲成：

| 方法       | 意义                       |
| ---------- | -------------------------- |
| 草图单模态 | 只依赖空间结构，不用语音   |
| 语音单模态 | 只依赖语义顺序，不用草图   |
| 早期融合   | 简单合并草图和语音         |
| 贪心修复   | 局部补边补点               |
| MMSB-Graph | 显式图编辑桥，全局约束路径 |

结果解释可以这样说：

草图单模态边 F1 只有 0.387，说明仅靠草图很难恢复完整关系；语音单模态边 F1 提升到 0.564，说明语音确实提供了关系和顺序信息；早期融合节点 F1 提升到 0.929，但边 F1 只有 0.577，说明简单融合仍会保留方向错误和结构残缺；贪心修复边 F1 达到 0.842，但仍不是全局最优；MMSB-Graph 在可控设定下达到节点 F1、边 F1、方向准确率均为 1.000，归一化 GED 和修改成本为 0。

然后补一句：

> **这个结果不表示真实世界已经完全解决，而是说明：如果我们能获得合理的草图图、语音图和候选融合图，图编辑桥可以作为一个有效的结构修复和诊断模块。**

这就是“真实完整”的讲法。

> 小样本评测协议与可控验证

内容：

* 24 个知识草图样例；
* 覆盖线性流程、多分支融合、反馈闭环、模型架构、实验流程；
* 对比草图单模态、语音单模态、早期融合、贪心修复、MMSB-Graph；
* 报告节点 F1、边 F1、GED、方向准确率、修改成本。

结论句：

> **课程阶段重点验证任务定义、指标链路和图编辑桥行为，而非声称大规模真实 SOTA。**

## 第 6 页：结果、局限与未来工作

标题：

> 结果说明与真实边界

内容：

* 语音相对草图提升边恢复能力；
* 早期融合提升节点覆盖，但仍有方向错误；
* 图编辑桥降低 GED 和修改成本；
* 当前结果是合成条件图可控验证；
* 后续需要真实 ASR/VLM、更多真人草图、用户修改轨迹、语义嵌入图搜索。

结论句：

> **我们完成了从产品原型到科研任务定义、方法建模、样例标注和评测协议的最小闭环。**
